# Exercício 01 — Isto é um algoritmo válido?

## Enunciado

Para cada situação abaixo, indique se é um algoritmo válido. Se não for, explique por quê.

Lembre dos três requisitos de um algoritmo:

- Finito
- Ordenado
- Sem ambiguidade

### Situações

**a.** "Some dois números."

**b.** "Leia um número. Se for positivo, mostre 'positivo'. Senão, mostre 'negativo'."

**c.** "Fique repetindo até acabar."

**d.** "Pegue um ingrediente qualquer e cozinhe."

---

# Resolução

### a. "Some dois números."

**Resposta:** ❌ Não é um algoritmo válido.

**Justificativa:**

A instrução é muito genérica. Não informa como os números serão obtidos nem onde o resultado será apresentado, tornando o processo ambíguo.

---

### b. "Leia um número. Se for positivo, mostre 'positivo'. Senão, mostre 'negativo'."

**Resposta:** ✅ É um algoritmo válido.

**Justificativa:**

Possui uma sequência lógica de passos, termina após a execução e não apresenta ambiguidades.

---

### c. "Fique repetindo até acabar."

**Resposta:** ❌ Não é um algoritmo válido.

**Justificativa:**

Não define uma condição de parada. A expressão "até acabar" é ambígua e pode gerar uma repetição infinita.

---

### d. "Pegue um ingrediente qualquer e cozinhe."

**Resposta:** ❌ Não é um algoritmo válido.

**Justificativa:**

Não especifica qual ingrediente utilizar nem como cozinhá-lo, permitindo diferentes interpretações.

---

## Conclusão

Para ser considerado válido, um algoritmo deve ser **finito**, **ordenado** e **sem ambiguidades**, garantindo que qualquer pessoa execute os mesmos passos e obtenha o resultado esperado.
