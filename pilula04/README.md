# Pílula #04 — Você já programa sem saber: algoritmo é receita de bolo

Tem uma notícia boa: você já sabe criar algoritmos. Faz isso o dia inteiro, só não chama por esse nome.

## O que é um algoritmo?
É uma sequência de passos clara e ordenada para resolver um problema. O exemplo mais perfeito do mundo é uma receita de bolo:
1. Bata 2 ovos
2. Misture 1 xícara de farinha
3. Adicione meia xícara de leite
4. Mexa até ficar homogêneo
5. Asse a 180°C por 30 minutos

Qualquer pessoa segue isso e chega no mesmo bolo. Isso é um algoritmo.

## As 3 regras de ouro
Todo bom algoritmo precisa ser:
* **Finito:** tem que terminar em algum momento.
* **Ordenado:** a ordem importa! Assar antes de misturar não vai dar certo.
* **Sem ambiguidade:** cada passo só pode ter uma interpretação.

## Você já faz isso o tempo todo
O trajeto para o trabalho, calcular o troco, montar um sanduíche, organizar a rotina da manhã — tudo isso são algoritmos que seu cérebro executa no piloto automático.

## Por que isso importa
Programar não é "aprender a falar com a máquina". É aprender a escrever essas receitas de forma tão precisa que uma máquina sem bom senso consiga seguir. A linguagem (Harbour, ADVPL) é só o idioma em que escrevemos a receita. O raciocínio vem primeiro.

---
*Micro-desafio mental: tente descrever, em passos numerados, como escovar os dentes — como se a pessoa nunca tivesse feito isso. Repare como você precisa ser bem mais detalhado do que imaginava. É exatamente assim que pensaremos ao programar.*
